## 🛡️ EvidenceGuard (에비던스 가드)

EvidenceGuard는 디지털 증거의 오염을 방지하고 무결성을 확보하기 위해 제작된 포렌식 보조 도구입니다.
'무결성이 깨진 증거는 증거가 아니다'라는 말을 보고 증거 지킴이 (Evidence Guard) 라고 이름붙였습니다.


## 🧐 왜 만들었나요?

포렌식 작업에서 가장 중요한 것은 '원본의 보존' 이라고 생각합니다. 분석을 위해 파일을 열거나 복제하는 과정에서 미세하게 변하는 타임스탬프와 데이터를 보호하고, 누가 봐도 믿을 수 있는 분석 보고서를 뽑아내보면 어떨까 하는 생각에 이 프로젝트를 시작했습니다.


## [작업 프로세스]
1. 원본 파일 선택
2. 포렌식 분석에 쓸 수 있는 암호화된 파일과 해시값 생성 (분석용 증거 획득)
3. 해시가 일치하는지 무결성 검증 및 복호화 (동일성 확인)
4. 데이터 내의 패턴을 분석해 증거 요소를 탐색
5. 보고서로 출력 (문서화)


## 🚀 핵심 포인트

* **꼼꼼한 무결성 검증 (Dual-Hashing):** 단일 해시의 취약점을 보완하기 위해 두 종류의 해시 알고리즘을 선택해 동시에 돌립니다. 두 값이 모두 일치해야 'PASS'를 띄웁니다.

* **포렌식 타임스탬프 복원:** 복호화된 파일은 운영체제에 의해 '수정된 날짜'가 바뀝니다. 이를 원본 파일의 메타데이터와 대조하여 강제로 원래대로(원본 파일의 메타데이터와 동일하게) 돌려놓는 기능을 넣었습니다.

* **정규식 검색:** 대용량 로그에서 이메일이나 전화번호를 찾을 때 버벅거리지 않도록 RegexOptions.Compiled를 적용했습니다. (반복적인 인터프리팅으로 인한 성능 저하를 최소화하기 위해 미리 기계어로 번역해두는 옵션)

* **리소스 증명:** 보고서에 당시 CPU 점유율과 RAM 가용 메모리량을 기록합니다. "분석 당시 시스템이 안정적이었다"는 것을 객관적으로 입증하기 위한 사항입니다.


## 🏗️ 인터페이스 기반의 유연한 설계 (Decoupled Architecture)

단순히 기능을 한 클래스에 몰아넣지 않고, 모든 서비스를 인터페이스와 구현체로 분리해 설계해보았습니다.

* **관심사의 분리:** 암호화, 해싱, 검색, 리포팅 등 각 기능을 독립적인 서비스 클래스로 캡슐화했습니다. 이로 인해 특정 로직(예: AES 암호화 방식)을 수정하더라도 다른 기능이나 UI 로직에 영향을 주지 않게 했습니다.

* **의존성 주입:** Prism의 IoC 컨테이너를 통해 필요한 서비스를 뷰모델에 주입합니다. 이렇게 하면 추후 특정 서비스를 교체(예: 다른 라이브러리를 사용한 PDF 엔진 등)해야 할 때도 인터페이스만 유지한 채 구현체만 바꾸면 됩니다.
또한 실제 파일 시스템이나 하드웨어에 의존하지 않고도, 가짜(Mock) 객체를 주입하여 비즈니스 로직만 독립적으로 검증하기도 훨씬 쉽습니다(xUnit 등의 테스트 프레임워크와 모킹 라이브러리(Moq 등)를 활용한 Unit Test 시).

* **로직 투명성:** 포렌식 도구의 특성상 로직의 투명성이 중요하므로, 각 인터페이스가 명확한 역할을 수행하도록 설계했습니다.


## 🛠 기술 스택 & 라이브러리

* **C# / WPF / Prism (MVVM):** 체계적인 의존성 주입(DI)을 바탕으로 유지보수가 편하도록 구조를 세팅했습니다.

* **iText :** PDF 보고서 생성 시 활용한 라이브러리입니다. 한글 깨짐 방지(Identity-H) 설정을 세팅했습니다.

* **Cryptography:** .NET 표준 라이브러리로 안전하게 AES 암호화를 구현했습니다.


## 📂 구조 맛보기

* **Core/Interface:** 설계 의도를 먼저 정의했습니다.

* **Services:** 실제 포렌식 로직 구현부입니다.

* **ViewModels:** 뷰와 로직을 Prism의 DelegateCommand 를 매개체로 연결하였습니다.


## 💡 트러블 슈팅 및 보완

* **한글 엑셀 인코딩 문제:** CSV 저장 시 UTF-8 with BOM 처리를 통해 엑셀에서 한글이 깨지는 현상을 해결했습니다.

* **싱글톤(Singleton) 관리:** 해시 및 암호화 서비스처럼 자원을 많이 쓰는 서비스는 싱글톤으로 등록하여 자원 낭비를 줄이고 데이터 일관성을 확보했습니다.

* **사용자 편의성(비프음):** 대용량 작업 완료 시 시각적 알림 외에도 System.Console.Beep을 활용해 사용자가 즉각 인지할 수 있도록 배려했습니다.


## 🎈 확장성 및 발전 가능성

* **상용 라이브러리 호환성:** 인터페이스 기반 설계를 바탕으로, 실무 환경에 맞춰 `DevExpress GridControl` 등 대용량 데이터 그리드로 교체하여 시각화 성능 극대화를 지향할 수 있습니다.
* **비동기 성능 최적화:** 기본적으로 async - await 등 비동기 처리가 되어 있으며, 현재 구조에 가상화 기술 등을 심화 적용한다면 수백만 건의 포렌식 데이터 로딩 시에도 UI 끊김 없는 분석 환경을 구축할 수 있는 확장성을 갖추고 있습니다.
